# CI Checks GitHub Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A GitHub Actions workflow gating PRs to `main` on typecheck + Expo health checks, enforced as a required status check via the existing `protect-main` ruleset.

**Architecture:** One workflow (`.github/workflows/ci.yml`) with a single job `checks` running frozen install → typed-route generation → `tsc --noEmit` → `expo-doctor`. The repo's existing branch ruleset (id 18800808) gains a `required_status_checks` rule requiring the `checks` context. Spec: `docs/superpowers/specs/2026-07-11-ci-checks-design.md`.

**Tech Stack:** GitHub Actions, Bun 1.3.14, Expo SDK 57 CLI, TypeScript ~6.0.3, `gh` CLI (repo admin).

## Global Constraints

- Working directory (git worktree — run everything here): `/Users/lukas/my-runner/.claude/worktrees/e2e-test-setup`
- Repo: `ljukas/my-runner`, default branch `main`, current branch `worktree-e2e-test-setup`, ruleset `protect-main` id `18800808`
- Package manager is Bun (`bun.lock` is the only lockfile); pin version `1.3.14`
- The job id MUST be `checks` with no `name:` override — the ruleset requires that exact context string
- The workflow MUST NOT have a `paths:` filter (required check + paths filter = PRs stuck on "Expected — waiting for status")
- Commit messages and PR titles follow Conventional Commits (PRs are squash-merged)
- 1Password commit signing can transiently fail with "failed to fill whole buffer" — retry the commit once; never disable signing
- Do NOT merge the PR at the end — verification stops at "check runs, is required, goes green"; merging is the user's call

### Task 2: CI workflow file

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: `bun run typecheck` script and `.bun-version` from Task 1
- Produces: check-run context `checks` (required by Task 3's ruleset rule)

- [ ] **Step 1: Rehearse the exact CI steps locally (the test)**

Run each, in order:

```bash
bun ci
bunx expo customize tsconfig.json
bun run typecheck
bunx expo-doctor
```

Expected: all exit 0. `expo customize tsconfig.json` prints that it generated/updated the file; `expo-doctor` ends with `20/20 checks passed. No issues detected!` (count may drift slightly with CLI updates; what matters is exit 0).

- [ ] **Step 2: Undo local side effects of the rehearsal**

`expo customize` rewrites `tsconfig.json` (reformats/strips comments) and generates gitignored `.expo/` files. Restore the tracked file:

```bash
git diff --stat tsconfig.json   # note if it changed
git checkout -- tsconfig.json
git status --short              # expect: clean (`.expo/` is gitignored)
```

- [ ] **Step 3: Write `.github/workflows/ci.yml`**

Exact content:

```yaml
name: CI

on:
  pull_request:
  push:
    branches: [main]

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: read

jobs:
  checks:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v5

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version-file: .bun-version

      - name: Install dependencies (frozen)
        run: bun ci

      - name: Generate expo-router typed routes
        run: bunx expo customize tsconfig.json

      - name: Typecheck
        run: bun run typecheck

      - name: Expo doctor
        run: bunx expo-doctor
```

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions checks workflow (typecheck, expo-doctor)"
```
