# Remove react-native-web / web platform support

**Date:** 2026-07-11
**Status:** Approved
**Branch:** `worktree-uniwind-styling` (worktree) — deliverable is a PR to `main`

## Goal

The app is mobile-only (iOS/Android) at this stage. Remove all react-native-web
code, web-specific platform forks, web config, and web-referencing starter
content so the codebase reads as if web support never existed. `expo start`
must no longer offer a working web target.

## Non-goals

- No ESLint guardrails against reintroducing web (YAGNI — once the dependency
  and config are gone, nothing pulls web back in).
- No broader rewrite of the starter screens; they are replaced later by the
  C25K features. Only web-related content is touched.
- `expo-web-browser` is **kept**: despite the name it provides the native
  in-app browser used by `external-link.tsx`.

## Approach

Full removal **plus** simplification of indirection that existed only because
of web (chosen over a conservative delete-files-only variant):

- The `use-color-scheme` hook was a one-line re-export whose only purpose was
  enabling a `.web.ts` fork — collapse it.
- The `EXPO_OS !== 'web'` guard in `external-link.tsx` is always true on
  native — make the in-app-browser path unconditional.

## Changes

### 1. Files deleted outright

| File | Reason |
|---|---|
| `src/components/app-tabs.web.tsx` | Web tab bar; native `app-tabs.tsx` remains |
| `src/components/animated-icon.web.tsx` | Web splash/icon variant (renders a `<div>`) |
| `src/components/animated-icon.module.css` | Only imported by the deleted `.web.tsx` |
| `src/hooks/use-color-scheme.web.ts` | Web static-render hydration workaround |
| `src/hooks/use-color-scheme.ts` | Re-export existing only to host the `.web.ts` fork |
| `src/components/web-badge.tsx` | Only rendered behind `Platform.OS === 'web'` |
| `assets/images/favicon.png` | Referenced only by app.json `web` block |
| `assets/images/tutorial-web.png` | Used only by the web-support tutorial section |
| `assets/images/expo-badge.png`, `assets/images/expo-badge-white.png` | Used only by `web-badge.tsx` |

### 2. Edits to shared files

- `src/app/index.tsx` — remove the `Platform.OS === 'web'` branch in
  `getDevMenuHint()`; remove `WebBadge` import and render.
- `src/app/explore.tsx` — remove the `web:` key from the `Platform.select`
  padding (android/undefined behavior unchanged); remove `WebBadge` import and
  render; remove the entire "Android, iOS, and web support" collapsible
  (including the `tutorial-web.png` image); remove the `web:` key from the
  SymbolView icon name map.
- `src/components/ui/collapsible.tsx` — remove the `web:` key from the
  SymbolView icon name map. Confirm against expo-symbols v57 types that the
  map type-checks without a `web` key.
- `src/components/external-link.tsx` — remove the `process.env.EXPO_OS`
  guard and `target="_blank"`; always `preventDefault()` and open via
  `openBrowserAsync`.
- `src/hooks/use-theme.ts` — import `useColorScheme` from `react-native`
  directly (hook file is deleted).
- `src/constants/theme.ts` — remove the `web:` branch from `Fonts`
  (`ios` + `default` remain; `default` covers Android).
- `src/global.css` — remove the top-level `:root` web font-stack block
  (`--font-display`/`--font-mono`/`--font-rounded`/`--font-serif` stacks were
  consumed only by `Fonts.web`; native `--font-mono` values come from the
  `@variant ios` / `@variant android` blocks inside `@layer theme`). Verify
  fonts visually in the simulator.

### 3. Config, dependencies, docs

- `package.json` — remove `react-native-web` and `react-dom` dependencies and
  the `"web": "expo start --web"` script; run `bun install` to sync `bun.lock`.
- `app.json` — remove the `"web"` block (`output: static`, favicon).
- `AGENTS.md` — update all web mentions: the app is iOS/Android only (line
  ~11), remove the `w` key / `bun run web` instructions (lines ~19–20), and
  rewrite the "Platform forks" architecture bullet — there are no `.web.tsx`
  forks anymore.
- `src/uniwind-types.d.ts` is auto-regenerated by Metro; no manual edit.

## Error handling / risks

- **SymbolView name map without `web` key fails to type-check** → check the
  v57 type first; if the key is required, keep the map minimal with whatever
  the type demands and note it in the PR.
- **Tailwind `font-mono`/font utilities break after removing the `:root`
  block** → Tailwind v4 defaults define font variables; the `@variant`
  overrides supply native values. Caught by the simulator visual check; if
  fonts regress, reintroduce only the needed variables inside `@layer theme`.
- **`react-dom` removal breaks Metro resolution** → nothing in the native
  graph imports it; caught by the Metro bundle during the simulator run.
