# Remove Web Platform Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove all react-native-web / web platform code, config, dependencies, and web-referencing starter content so the app is iOS/Android only.

**Architecture:** Pure removal + simplification per the approved spec (`docs/superpowers/specs/2026-07-11-remove-web-support-design.md`). Delete web-only platform-fork files, strip `web` branches from shared files, collapse indirection that existed only for web (the `use-color-scheme` hook, the `EXPO_OS` guard in `external-link`), remove web deps/config, update docs. No test runner exists in this repo; each task's verify cycle is `bunx tsc --noEmit` plus targeted greps, with a final iOS-simulator run-through.

**Tech Stack:** Expo SDK 57, React Native 0.86, React 19.2, TypeScript ~6.0, expo-router, Uniwind (Tailwind v4), Bun.

## Global Constraints

- Package manager is **Bun**: `bun install`, `bunx tsc --noEmit`, `bun run lint`. Never use npm/yarn/pnpm.
- Work in this worktree (`/Users/lukas/my-runner/.claude/worktrees/uniwind-styling`), branch `worktree-uniwind-styling`. Do not `cd` to the main checkout.

- `withUniwindConfig` must stay the outermost wrapper in `metro.config.js` (this plan does not touch that file).
- Dev-server ports **8081 and 8082 belong to the user's own processes — never bind to them or kill them**. Verification Metro uses port **8090**.
- No destructive/force git commands (`push --force`, `reset --hard`, bare `git stash`, ...).

- [ ] **Step 1: Rewrite `src/components/external-link.tsx`**

```tsx
import { Href, Link } from 'expo-router';
import { openBrowserAsync, WebBrowserPresentationStyle } from 'expo-web-browser';
import { type ComponentProps } from 'react';

type Props = Omit<ComponentProps<typeof Link>, 'href'> & { href: Href & string };

export function ExternalLink({ href, ...rest }: Props) {
  return (
    <Link
      {...rest}
      href={href}
      onPress={async (event) => {
        // Open the link in an in-app browser instead of the system browser.
        event.preventDefault();
        await openBrowserAsync(href, {
          presentationStyle: WebBrowserPresentationStyle.AUTOMATIC,
        });
      }}
    />
  );
}
```
